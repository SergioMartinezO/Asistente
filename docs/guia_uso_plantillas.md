# Guía rápida de uso — Plantillas para proyectos electrónicos

Esta guía te ayuda a elegir y usar correctamente las plantillas del proyecto para obtener respuestas completas, no vagas y listas para ejecución.

---

## 1) ¿Qué plantilla debo usar?

- Usa `plantilla_prompt_diseno_electronico.md` si necesitas diseñar y construir el producto.
- Usa `plantilla_prompt_diseno_electronico_costos_riesgos.md` si además necesitas costos, riesgos y KPIs.
- Usa `plantilla_licitacion_propuesta_electronica.md` si necesitas propuesta formal para cliente/licitación.

> Índice general: `plantillas_indice.md`

---

## 2) Campos mínimos que debes completar siempre

Para evitar respuestas genéricas, completa como mínimo:

- nombre del proyecto,
- objetivo funcional,
- fecha de inicio (`YYYY-MM-DD`),
- restricciones técnicas (energía, entorno, conectividad),
- presupuesto tope (si aplica),
- nivel de detalle esperado de entregables.

---

## 3) Estructura de salida que debes exigir

Incluye explícitamente estos puntos:

1. Cronograma por fases con inicio/fin.
2. Entregables verificables por fase.
3. Componentes con justificación técnica.
4. Código con estándares de desarrollo.
5. Diagramas técnicos (bloques, circuito, flujo).
6. Artefactos finales (Word + Web + código + diagramas).
7. Confirmación final de viabilidad.

---

## 4) Checklist de control de calidad

Antes de aceptar la respuesta, revisa:

- [ ] ¿Cada fase tiene fechas de inicio y fin?
- [ ] ¿Los entregables son concretos y medibles?
- [ ] ¿Se incluyeron costos/riesgos cuando se solicitaron?
- [ ] ¿Hay viabilidad técnica, económica y operativa?
- [ ] ¿Cierra con una confirmación clara de ejecución?

---

## 5) Error común y cómo evitarlo

**Error común:** pedir “diseña un dispositivo” sin restricciones ni fecha.

**Cómo evitarlo:** agrega siempre fecha de inicio, límites de costo y criterio de aceptación por fase.
